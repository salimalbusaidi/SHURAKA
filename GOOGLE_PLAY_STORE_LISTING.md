## ١٠. App Signing & Build Instructions

### خطوات البناء والنشر:

```bash
# 1. تثبيت EAS CLI
npm install -g eas-cli

# 2. تسجيل الدخول في Expo
eas login

# 3. إعداد مشروع EAS
eas build:configure

# 4. بناء AAB للإنتاج (Google Play)
eas build --platform android --profile production

# 5. بناء APK للاختبار
eas build --platform android --profile preview
```

### إعداد eas.json:
```json
{
  "cli": { "version": ">= 5.0.0" },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "android": {
        "buildType": "apk"
      }
    },
    "production": {
      "android": {
        "buildType": "app-bundle"
      }
    }
  },
  "submit": {
    "production": {}
  }
}
```

### رفع على Google Play Console:
1. افتح [Google Play Console](https://play.google.com/console)
2. أنشئ تطبيقًا جديدًا
3. اذهب إلى: Release > Production > Create new release
4. ارفع ملف `.aab` الناتج من EAS Build
5. أضف Release Notes بالعربية
6. أكمل Data Safety + Content Rating
7. راجع واضغط "Review release"

## ١١. متغيرات البيئة المطلوبة (Production)

```bash
# Supabase / OnSpace Cloud
EXPO_PUBLIC_SUPABASE_URL=https://xxxxx.backend.onspace.ai
EXPO_PUBLIC_SUPABASE_ANON_KEY=eyJ...

# Stripe (في Edge Functions فقط - لا تضعها في Frontend)
# STRIPE_SECRET_KEY → في OnSpace Cloud Secrets
# STRIPE_WEBHOOK_SECRET → في OnSpace Cloud Secrets
# STRIPE_STARTER_PRICE_ID → في OnSpace Cloud Secrets
# STRIPE_GROWTH_PRICE_ID → في OnSpace Cloud Secrets
# STRIPE_PARTNERS_PLUS_PRICE_ID → في OnSpace Cloud Secrets
```

⚠️ **تنبيه أمني:** لا تضع أي مفتاح سري (Secret Key) في ملف `.env` الخاص بالواجهة الأمامية.
جميع المفاتيح السرية يجب أن تكون في OnSpace Cloud Secrets وتُستخدم فقط من Edge Functions.
